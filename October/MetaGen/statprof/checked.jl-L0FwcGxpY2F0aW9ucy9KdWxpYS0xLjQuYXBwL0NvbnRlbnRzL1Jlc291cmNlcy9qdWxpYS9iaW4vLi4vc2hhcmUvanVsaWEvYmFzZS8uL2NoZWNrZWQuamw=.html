35 # LLVM has several code generation bugs for checked integer arithmetic (see e.g.
36 # #4905). We thus distinguish between operations that can be implemented via
37 # intrinsics, and operations for which we have to provide work-arounds.
38
39 # Note: As far as this code has been tested, most checked_* functions are
40 # working fine in LLVM. (Note that division is still handled via `base/int.jl`,
41 # which always checks for overflow, and which provides its own sets of
42 # work-arounds for LLVM codegen bugs.) However, the comments in `base/int.jl`
43 # and in issue #4905 are more pessimistic. For the time being, we thus retain
44 # the ability to handle codegen bugs in LLVM, until the code here has been
45 # tested on more systems and architectures. It also seems that things depend on
46 # which compiler that was used to build LLVM (i.e. either gcc or clang).
47

255 f = if y > 0
